This guide will help you set up the DPM Delivery API, configure your environment, and make your first API calls.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 16.x or higher
- **PostgreSQL** 12.x or higher
- **npm** or **yarn** package manager
- **Git** for version control

You'll also need accounts for these third-party services:

- **Cloudinary** - For file uploads and image management
- **Arkesel** or **Twilio** - For SMS notifications
- **Kowri** or **Paystack** - For payment processing

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the DPM Delivery API repository from GitHub:

    ```bash
    git clone https://github.com/Felix-Asante/DPM-delivery-api.git
    cd DPM-delivery-api
    ```
  </Step>

  <Step title="Install dependencies">
    Install the required Node.js packages:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and configure your settings:

    ```bash
    cp .env.example .env
    ```

    Open `.env` in your editor and configure the required variables:

    ```bash
    # Server Configuration
    PORT=8080
    NODE_ENV=development

    # Database Configuration
    TYPEORM_HOST=localhost
    TYPEORM_PORT=5432
    TYPEORM_USERNAME=your_username
    TYPEORM_PASSWORD=your_password
    TYPEORM_DATABASE=dpm_delivery

    # JWT Secret
    JWT_SECRET=your-super-secret-jwt-key

    # Cloudinary Configuration
    CLOUDINARY_CLOUD_NAME=your_cloud_name
    CLOUDINARY_API_KEY=your_api_key
    CLOUDINARY_SECRET=your_api_secret

    # SMS Service (Arkesel)
    ARKESEL_KEY=your_arkesel_api_key
    ARKESEL_ENDPOINT_V2=https://sms.arkesel.com/api/v2

    # Payment Gateway (Kowri)
    KOWRI_APP_ID=your_app_id
    KOWRI_APP_SECRET=your_app_secret
    KOWRI_EP=https://api.kowri.com

    # Application URLs
    FRONTEND_URL=http://localhost:3000
    DASHBOARD_URL=http://localhost:3001
    ADMIN_PHONE_NUMBER=+233241234567
    RIDER_DEFAULT_PASSWORD=Rider123!
    ```

    <Note>
      See the [Environment Setup](/guides/environment-setup) guide for detailed configuration of each variable.
    </Note>
  </Step>

  <Step title="Set up the database">
    Create your PostgreSQL database:

    ```bash
    createdb dpm_delivery
    ```

    Run the database migrations to create the schema:

    ```bash
    npm run migration:run
    ```

    <Accordion title="Migration output">
      ```
      query: SELECT * FROM current_schema()
      query: SELECT version FROM "typeorm_migrations" ...
      0 migrations are already loaded in the database.
      1 migrations were found in the source code.
      Migration UserEntity1234567890123 has been executed successfully.
      ```
    </Accordion>
  </Step>

  <Step title="Start the development server">
    Launch the API server in development mode:

    <CodeGroup>
      ```bash npm
      npm run start:dev
      ```

      ```bash yarn
      yarn start:dev
      ```
    </CodeGroup>

    The API will be available at `http://localhost:8080/api/v1/`

    <Info>
      The Swagger API documentation is automatically available at `http://localhost:8080/api/v1/doc`
    </Info>
  </Step>
</Steps>

## Make your first API call

Now that the server is running, let's make some API calls to test the setup.

### 1. Create a user account

Register a new user account:

<CodeGroup>
  ```bash cURL
  curl -X POST http://localhost:8080/api/v1/auth/signup \
    -H "Content-Type: application/json" \
    -d '{
      "phone": "+233241234567",
      "password": "SecurePass123!",
      "firstName": "John",
      "lastName": "Doe",
      "role": "USER"
    }'
  ```

  ```javascript JavaScript
  const response = await fetch('http://localhost:8080/api/v1/auth/signup', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      phone: '+233241234567',
      password: 'SecurePass123!',
      firstName: 'John',
      lastName: 'Doe',
      role: 'USER'
    })
  });

  const data = await response.json();
  console.log(data);
  ```

  ```python Python
  import requests

  response = requests.post(
      'http://localhost:8080/api/v1/auth/signup',
      json={
          'phone': '+233241234567',
          'password': 'SecurePass123!',
          'firstName': 'John',
          'lastName': 'Doe',
          'role': 'USER'
      }
  )

  print(response.json())
  ```
</CodeGroup>

### 2. Verify your account

Use the verification code sent to verify your account:

```bash cURL
curl -X GET http://localhost:8080/api/v1/auth/verify-account/123456
```

### 3. Login and get an access token

Login with your credentials to receive a JWT token:

<CodeGroup>
  ```bash cURL
  curl -X POST http://localhost:8080/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "phone": "+233241234567",
      "password": "SecurePass123!"
    }'
  ```

  ```javascript JavaScript
  const response = await fetch('http://localhost:8080/api/v1/auth/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      phone: '+233241234567',
      password: 'SecurePass123!'
    })
  });

  const { token } = await response.json();
  ```
</CodeGroup>

### 4. Create a shipment (authenticated request)

Use the JWT token to make an authenticated request:

```bash cURL
curl -X POST http://localhost:8080/api/v1/shipping \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "sender_name": "John Doe",
    "sender_phone": "+233241234567",
    "sender_address": "123 Main St, Accra",
    "recipient_name": "Jane Smith",
    "recipient_phone": "+233241234568",

    "package_value": 500.00,
    "shipment_option": "standard",
    "mode_of_shipment": "Bike",
    "city": "Nkawkaw"
  }'
```

## Explore the Swagger documentation

The API includes interactive Swagger documentation where you can test all endpoints:

1. Open `http://localhost:8080/api/v1/doc` in your browser
2. Click **Authorize** and enter your JWT token
3. Try out any endpoint with the "Try it out" button

<Tip>
  The Swagger UI persists your authorization token, so you only need to enter it once per session.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection error">
    If you see `error: password authentication failed for user`, verify your PostgreSQL credentials in `.env`:

    ```bash
    TYPEORM_USERNAME=your_username
    TYPEORM_PASSWORD=your_password
    TYPEORM_HOST=localhost
    ```

    Ensure PostgreSQL is running:
    ```bash
    # On macOS with Homebrew
    brew services start postgresql

    # On Ubuntu/Debian
    sudo systemctl start postgresql
    ```
  </Accordion>

  <Accordion title="Port already in use">
    If port 8080 is already in use, change the `PORT` variable in your `.env` file:

    ```bash
    PORT=3000
    ```

    Then restart the server.
  </Accordion>

  <Accordion title="JWT authentication fails">
    If you receive `401 Unauthorized` errors:

    1. Verify your JWT token is valid and not expired
    2. Ensure you're including the token in the Authorization header:
       ```
       Authorization: Bearer YOUR_JWT_TOKEN
       ```
    3. Check that your `JWT_SECRET` in `.env` matches the one used to sign the token
  </Accordion>

  <Accordion title="Phone verification code not received">
    In development mode, the verification code is returned in the signup response. Check your SMS service configuration:

    - Verify `ARKESEL_KEY` is correct
    - Ensure the phone number format is valid (E.164 format with country code)
    - Check the Arkesel dashboard for SMS delivery status
  </Accordion>
</AccordionGroup>
