## Endpoint

```
POST /api/v1/auth/signup
```

Creates a new user account and sends a verification code via SMS to the provided phone number. The account must be verified using the verification code before it can be used.

## Request Body

<ParamField body="fullName" type="string" required>
  The full name of the user
  
  **Validation**: Must be a non-empty string (`@IsString`)
</ParamField>

<ParamField body="phone" type="string" required>
  The phone number of the user in Ghana format
  
  **Validation**: Must be a valid phone number string (`@IsString`)
  
  **Note**: Phone numbers must be unique in the system
</ParamField>

<ParamField body="password" type="string" required>
  The password for the account
  
  **Validation**: Must be a non-empty string (`@IsString`)
  
  **Note**: Passwords are automatically hashed using bcrypt with 10 salt rounds before storage
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the created user
</ResponseField>

<ResponseField name="fullName" type="string">
  Full name of the user
</ResponseField>

<ResponseField name="phone" type="string">
  Phone number of the user
</ResponseField>

<ResponseField name="isVerified" type="boolean">
  Account verification status (default: `false`)
</ResponseField>

<ResponseField name="code" type="string">
  Verification code sent via SMS
</ResponseField>

<ResponseField name="codeUseCase" type="string">
  Use case for the code (value: `"ACTIVATE_ACCOUNT"`)
</ResponseField>

<ResponseField name="codeExpiryDate" type="string">
  ISO 8601 timestamp when the verification code expires
</ResponseField>

<ResponseField name="role" type="object">
  User role information
  
  <ResponseField name="name" type="string">
    Role name (default: `"USER"`)
  </ResponseField>
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of account creation
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

## Implementation Details

**Source**: `src/auth/auth.controller.ts:45-51`

**DTO**: `CreateUserDto` from `src/users/dto/create-user.dto.ts`

**Swagger Decorators**:
- `@ApiBadRequestResponse({ description: 'Bad request' })`
- `@ApiInternalServerErrorResponse({ description: 'Internal server error' })`
- `@ApiCreatedResponse({ description: 'user created successfully' })`

**Process Flow**:
1. Validates that phone number is not already registered
2. Generates a 6-digit OTP verification code
3. Sets code expiry date
4. Assigns default USER role
5. Hashes password using bcrypt (10 salt rounds)
6. Saves user to database
7. Sends verification code via SMS
8. Returns user object with verification code

## Next Steps

After signup, the user must verify their account using the verification code sent via SMS. See the [Verify Account](/api/auth/verify-account) endpoint.
