## Overview

The Reviews API provides functionality for managing customer reviews and ratings. Reviews are tightly integrated with the booking system - customers can rate their booking experience, and these ratings are automatically reflected in the place's overall rating.

## Authentication

Most endpoints require JWT authentication via Bearer token:
- **GET /reviews/:id**: No authentication required (public)
- **DELETE /reviews/:id**: Requires authentication (ADMIN or review owner)
- **POST /bookings/:id/rate**: Requires USER role

---

## Create Review (Rate Booking)

<CodeGroup>
```bash cURL
curl -X POST https://api.dpm-delivery.com/bookings/{bookingId}/rate \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "rating": 5,
    "comment": "Excellent service and great location!"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.dpm-delivery.com/bookings/{bookingId}/rate', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    rating: 5,
    comment: 'Excellent service and great location!'
  })
});
```
</CodeGroup>

**Endpoint:** `POST /bookings/:id/rate`

**Required Role:** USER

**Description:** Allows users to rate and review their booking experience. The review is automatically associated with the booking and the place rating is updated.

**Path Parameters:**
<ParamField path="id" type="string" required>
  The UUID of the booking to rate
</ParamField>

**Request Body:**
<ParamField body="rating" type="number" required>
  Rating value between 1 and 5 (inclusive)
</ParamField>

<ParamField body="comment" type="string" required>
  Written review comment describing the experience
</ParamField>

**Response:**
```json
{
  "id": "uuid",
  "rating": 5,
  "comment": "Excellent service and great location!",
  "date": "2026-03-04T10:00:00Z",
  "user": {
    "id": "uuid",
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com"
  },
  "booking": {
    "id": "uuid",
    "place": {
      "id": "uuid",
      "name": "Beach Resort",
      "rating": 4.7
    }
  },
  "createdAt": "2026-03-04T10:00:00Z",
  "updatedAt": "2026-03-04T10:00:00Z"
}
```

<Note>
  The place's overall rating is automatically recalculated when a new review is submitted.
</Note>

## Delete Review

<CodeGroup>
```bash cURL
curl -X DELETE https://api.dpm-delivery.com/reviews/{id} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.dpm-delivery.com/reviews/{id}', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});
```
</CodeGroup>

**Endpoint:** `DELETE /reviews/:id`

**Required Role:** ADMIN or review owner (USER)

**Path Parameters:**
<ParamField path="id" type="string" required>
  The UUID of the review to delete
</ParamField>

**Response:**
```json
{
  "success": true,
  "message": "The review with id: {uuid} has been deleted"
}
```

<Warning>
  When a review is deleted, the place's overall rating is automatically recalculated. Users can only delete their own reviews unless they have ADMIN role.
</Warning>

## Integration with Bookings

Reviews are tightly integrated with the booking system:

### One-to-One Relationship
- Each review is associated with exactly one booking
- Each booking can have at most one review
- Users can only review bookings they have made

### Automatic Rating Calculation

When a review is created or deleted, the place's overall rating is automatically updated:

**On Review Creation:**
```
newRating = ((currentRating × reviewCount) + newReviewRating) / (reviewCount + 1)
```

**On Review Deletion:**
```
newRating = ((currentRating × reviewCount) - deletedReviewRating) / (reviewCount - 1)
```

If the last review is deleted, the place's rating is set to `null`.

## Validation Rules

### Rating Validation
- Must be a number
- Minimum value: 1
- Maximum value: 5
- Can include decimal values (e.g., 4.5)

### Comment Validation
- Required field
- Must be a string
- No explicit length limits (configured at database level)

### Authorization Rules

**Creating Reviews:**
- User must be authenticated
- User must have USER role
- User must own the booking being reviewed
- Booking must exist and be valid

**Deleting Reviews:**
- User must be authenticated
- User must be either:
  - The review owner (USER role)
  - System administrator (ADMIN role)

## Common Use Cases

### Submitting a Review After Stay

1. User completes their booking/stay
2. User calls `POST /bookings/{bookingId}/rate` with rating and comment
3. System creates review and updates place rating
4. Review is now visible to other users viewing the place

### Moderating Inappropriate Reviews

1. Admin identifies inappropriate review
2. Admin calls `DELETE /reviews/{reviewId}`
3. System removes review and recalculates place rating

### Displaying Reviews on Place Page

1. Fetch place details (which includes overall rating)
2. Fetch bookings for the place
3. Reviews are included in booking objects (eager loaded)
4. Display reviews with user information and ratings
