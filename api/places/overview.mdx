## Introduction

The Places API allows you to manage vendor locations (restaurants, stores, service providers) in the DPM Delivery platform. Places represent the core business entities that offer products and services to customers.

## Multi-Tenant Architecture

Each place operates as a semi-independent tenant with:

- **Place Admin**: Dedicated administrator with `PLACE_ADMIN` role
- **Product Categories**: Custom menu/product organization per place
- **Opening Hours**: Flexible schedule management
- **Location Data**: Geospatial coordinates for proximity search
- **Independent Products**: Each place manages its own product catalog

## Base URL

```
POST   /places
GET    /places
GET    /places/:id
PUT    /places/:id
DELETE /places/:id
```

## Key Features

### Geospatial Search

Find places near users based on latitude/longitude coordinates:

```http
GET /places/near/me?latitude=5.6037&longitude=-0.1870
```

### Category Filtering

Filter places by business category (restaurants, pharmacies, etc.):

```http
GET /places?category={categoryId}
```

### Place Discovery

- **Search**: Full-text search with optional geolocation
- **Popular Places**: Trending locations based on visits
- **New Places**: Recently added vendors
- **Nearby Places**: Distance-based recommendations

## Permissions

- **ADMIN**: Full access to all place operations
- **PLACE_ADMIN**: Manage own place, products, and opening hours
- **USER**: View places, rate, like/unlike
- **Public**: Search, view details, nearby places
