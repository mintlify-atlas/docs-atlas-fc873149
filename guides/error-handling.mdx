## Overview

The DPM Delivery API uses NestJS's built-in exception handling along with custom error messages and validation pipes to provide consistent, informative error responses.

## Global Validation

### Validation Pipe Configuration

Global validation is configured in `src/main.ts` using class-validator:

```typescript
import { ValidationPipe } from '@nestjs/common';

app.useGlobalPipes(
  new ValidationPipe({
    whitelist: true,
    transform: true,
    forbidNonWhitelisted: true,
    transformOptions: {
      enableImplicitConversion: true,
    },
  }),
);
```

### Configuration Options Explained

<CardGroup cols={2}>
  <Card title="whitelist" icon="filter">
    Strips properties that don't have decorators in the DTO
  </Card>
  <Card title="transform" icon="wand-magic-sparkles">
    Automatically transforms payloads to DTO instances
  </Card>
  <Card title="forbidNonWhitelisted" icon="ban">
    Throws an error if non-whitelisted properties are present
  </Card>
  <Card title="enableImplicitConversion" icon="arrows-rotate">
    Automatically converts primitive types (string to number, etc.)
  </Card>
</CardGroup>

<Info>
  These settings ensure that only validated, typed data reaches your route handlers, preventing common security vulnerabilities.
</Info>

import { ApiProperty, ApiPropertyOptional } from '@nestjs/swagger';

  IsArray,
  IsInt,
  IsNumber,

## Error Response Format

### Standard Error Response

All errors follow NestJS's standard error format:

```json
{
  "statusCode": 400,
  "message": "Validation failed",
  "error": "Bad Request"
}
```

### Validation Error Response

Validation errors include detailed information:

```json
{
  "statusCode": 400,
  "message": [
    "place must be a string",
    "recipient_phone must be a valid phone number",
    "delivery_address should not be empty"
  ],
  "error": "Bad Request"
}
```

    }
    
    // Process rating...
  }
}

## Error Handling Examples

### Handle Not Found

```typescript
@Get(':id')
async findOne(@Param('id') id: string) {
  const place = await this.placesService.findOne(id);
  
  if (!place) {
    throw new NotFoundException(ERRORS.PLACES.NOT_FOUND);
  }
  
  return place;
}
```

### Handle Duplicate Entry

```typescript
@Post()
async create(@Body() createUserDto: CreateUserDto) {
  const existingUser = await this.usersService.findByEmail(
    createUserDto.email
  );
  
  if (existingUser) {
    throw new ConflictException(ERRORS.EMAIL_ALREADY_TAKEN.EN);
  }
  
  return this.usersService.create(createUserDto);
}
```

### Handle Authorization

```typescript
@Delete(':id')
async remove(
  @Param('id') id: string,
  @CurrentUser() user: User,
) {
  const review = await this.reviewsService.findOne(id);
  
  if (!review) {
    throw new NotFoundException(ERRORS.REVIEWS.NOT_FOUND.EN);
  }
  
  if (review.user.id !== user.id) {
    throw new ForbiddenException(ERRORS.REVIEWS.NOT_YOURS.EN);
  }
  
  return this.reviewsService.remove(id);
}
```

### Try-Catch for External APIs

```typescript
async checkPaymentStatus(transactionId: string) {
  try {
    const response = await axios.post(
      externalApis.payment.checkStatus(),
      { transactionId },
      { headers: { appId: this.appId } }
    );
    return response.data;
  } catch (error) {
    if (error.response?.status === 404) {
      throw new NotFoundException('Transaction not found');
    }
    if (error.response?.status >= 500) {
      throw new BadGatewayException('Payment service unavailable');
    }
    throw new InternalServerErrorException(
      'Failed to check payment status'
    );
  }
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Specific Exception Types">
    Choose the most appropriate exception type for the error:

    ```typescript
    // Good
    if (!user) {
      throw new NotFoundException('User not found');
    }
    if (user.id !== currentUser.id) {
      throw new ForbiddenException('Cannot access this resource');
    }

    // Avoid
    if (!user) {
      throw new BadRequestException('User not found'); // Wrong type
    }
    ```
  </Accordion>

  <Accordion title="Use Centralized Error Messages">
    Always use error messages from `ERRORS` constant:

    ```typescript
    // Good
    throw new NotFoundException(ERRORS.BOOKINGS.NOT_FOUND.EN);

    // Avoid
    throw new NotFoundException('Booking not found'); // Hardcoded
    ```
  </Accordion>

  <Accordion title="Validate Early">
    Use DTOs and validation pipes to catch errors early:

    ```typescript
    // Good - validation happens automatically
    @Post()
    async create(@Body() dto: CreateBookingDto) {
      return this.service.create(dto);
    }

    // Avoid - manual validation in service
    async create(data: any) {
      if (!data.place) throw new BadRequestException(...);
      if (!data.phone) throw new BadRequestException(...);
      // ...
    }
    ```
  </Accordion>

  <Accordion title="Log Internal Errors">
    Always log internal errors before throwing:

    ```typescript
    try {
      await externalService.call();
    } catch (error) {
      console.error('External service error:', error);
      throw new InternalServerErrorException(
        'Service temporarily unavailable'
      );
    }
    ```
  </Accordion>

  <Accordion title="Don't Expose Sensitive Information">
    Never expose internal details or secrets in error messages:

    ```typescript
    // Good
    throw new UnauthorizedException('Invalid credentials');

    // Avoid
    throw new UnauthorizedException(
      `Database connection failed: ${dbError.connectionString}`
    );
    ```
  </Accordion>
</AccordionGroup>

## Testing Error Handling

### Unit Test Example

```typescript
import { NotFoundException } from '@nestjs/common';
import { Test } from '@nestjs/testing';
import { BookingsService } from './bookings.service';
import { ERRORS } from 'src/utils/errors';

describe('BookingsService', () => {
  let service: BookingsService;

  beforeEach(async () => {
    const module = await Test.createTestingModule({
      providers: [BookingsService],
    }).compile();

    service = module.get<BookingsService>(BookingsService);
  });

  describe('findOne', () => {
    it('should throw NotFoundException when booking does not exist', async () => {
      jest.spyOn(service, 'findOne').mockRejectedValue(
        new NotFoundException(ERRORS.BOOKINGS.NOT_FOUND.EN)
      );

      await expect(service.findOne('invalid-id')).rejects.toThrow(
        NotFoundException,
      );
    });
  });
});
```
